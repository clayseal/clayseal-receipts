# Clay Seal — how it works, end to end

**Clay Seal binds an agent's authorization to what it actually executes, and makes both provable.** It runs as an enforcement gateway in front of an agent's tool calls: every side-effecting action is decided *before* it happens and recorded into a tamper-evident log afterward. What follows is the life of a single governed tool call — e.g. "issue a payroll bonus" — and how each control seams into the next.

**Identity comes first, and it is attested, not declared.** Before any action, the agent's workload presents a signed attestation document; selectors are derived from that verified evidence, and the matching registration entry — not the caller — dictates the agent's type, tenant, and scopes. The result is a short-lived credential (a SPIFFE-identified JWT-SVID, optionally a proof-of-possession capability token). The seam `authority_binding.py` normalizes this into an **AuthorityContext** — subject, tenant, issuer, capabilities, trust tier — that the runtime carries into every downstream decision. This matters because it means every enforcement check and every receipt is bound to a *verified* identity, never a self-asserted one; a compromised prompt cannot promote the agent's authority.

**At query start, the runtime scopes the session to its task — from the original request, captured before contamination.** From the user's opening prompt (taken *before* any retrieved document, memory note, or tool output can enter and bias it), Clay Seal derives a **capability lease**: which tools and which target entities are legitimately in play for this task. Entity references ("Camille Moreau") are resolved against the tenant directory; named groups expand, bounded, along membership edges. The lease is the agent's least-authority envelope for the session — a write to a tool or an entity outside it is refused later *even with otherwise-valid credentials*. This is what defeats connector-substitution (an ungated "legacy" path standing in for the governed one) and injection that aims an in-scope capability at an out-of-scope target.

**When the agent proposes the write, it must first obtain a commit token — and this is the core primitive.** The control plane issues a signed, single-use token bound to the *exact* tool name, a hash of the *exact* arguments, the tenant, the authority id, and the query id. The gateway re-derives that hash at the execution boundary and rejects any mismatch. This makes approval **payload-exact and non-repudiable**: if the agent gets a token for one payload and then executes a different one — a rounded amount, an added beneficiary, a substituted employee id — the `arguments_hash` no longer matches and the call is denied. Single-use consumption closes replay within the session. A sandbox **governor** sits alongside, enforcing per-tool policy — this tool requires a commit token, that one additionally requires human step-up, this class is denied outright.

**The token proves *this call*; two session-level controls prove the *aggregate* is legitimate, because a valid token per call is not enough.** First, a **cumulative value budget** sums the real quantity moved — dollars, headcount — across the whole session against a ceiling that is *a property of the requesting human's own authority* (the agent inherits not just *what* it may do but *how much* it may move — the same inheritance principle, extended to magnitude). It is deliberately tool-agnostic: the governed tool and any legacy connector debit the *same* budget, so the ceiling cannot be sidestepped by switching rails. This catches the aggregate over-authority payout whose every individual line looked fine, and the task fragmented into sub-threshold pieces to duck a review bar. Second, **effect supersession** keeps that from over-blocking honest work: a correction reissued under a shared idempotency key *replaces* rather than *adds*, so a legitimate same-session fix is not mistaken for accumulation — and it is safe against a dishonest agent by construction, since a replace can only ever *reduce* a total, never inflate it. A per-session **behavior signal** scores each action for anomaly (a side-effecting call after a read-only stretch, off-task references, repetition) and rides along as evidence.

**Every one of these decisions — allow, step-up, or deny, with reasons — is appended to a hash-chained audit log, and can be exported as a verifiable receipt.** The receipt carries the policy decision, the authority binding, and a Merkle **inclusion proof** (RFC 6962) against the append-only log, with consistency proofs and witness quorum for the log itself, issued as an **IETF SCITT** signed statement in **C2SP** checkpoint format. The consequence: an auditor, a compliance team, or the connected platform can verify *what an agent did and that it was authorized* without trusting Clay Seal's word — cryptographically, offline. This is the surface the IETF is standardizing right now (SCITT/SCRAPI, Agent Enforcement Receipts, Verifiable AI Provenance); Clay Seal already emits it, and — unlike attestation-only tooling — its receipts record an *enforced* decision, not merely an observed one.

**The organizing principle, and the roadmap.** Every authority signal originates *outside* the agent's own reasoning loop — attested identity, requester-inherited budgets, signed tokens — so no amount of adversarial prompting can manufacture authorization from within. The controls above are individually sound and composable; the near-term productionization is to route them through a single **Policy Decision Point** (PDP) with an AuthZEN-shaped interface — one evaluator, one explicit order, one typed decision that is exactly what gets receipted — so the enforcement path is legible in one contract, a downstream service can re-check the same decision (defense in depth, no self-authorization), and the behavior signal graduates from recorded evidence to a step-up trigger. The through-line: **loosen human-in-the-loop-per-action exactly where the binding underneath can now be trusted** — auto-execute what is verifiably in-scope, in-budget, and payload-bound; reserve human attention for the genuinely novel or over-threshold; and hand back a verifiable receipt for every action, either way.
