# Compliance field mapping (SOTA-4)

Receipt bundles already contain audit-grade evidence. This mapping layer projects
those fields onto **control families** compliance teams recognize, and emits
**SIEM-native** records for ingestion.

Crosswalk YAML: [`compliance/`](../compliance/)

## Supported profiles

| Profile | CLI value | Focus |
|---------|-----------|-------|
| EU AI Act | `eu-ai-act` | Logging duties, provenance, human oversight state |
| SOC 2 | `soc2` | CC6 authorization + CC7 monitoring inputs |
| ISO 27001 | `iso27001` | A.8.15/A.8.16 operational logging |

## Required evidence fields (all profiles)

Each profile validates that the receipt exposes:

- **system_version** — model provenance hash and/or SDK version
- **policy_reference** — policy commitment
- **input_commitment** — `execution_proof.context_hash`
- **output_commitment** — `execution_proof.output_hash`
- **automated_reasoning** — decision outcome / recommended action
- **human_oversight** or **event_actor** — approval state or authority/action (profile-specific)
- **integrity_protection** — verification state, assurance tier, and/or signatures

## CLI

Compliance-mapped export:

```bash
arctl audit-summary receipt.json --profile eu-ai-act
arctl format-bundle receipt.json --profile soc2 --out mapped.json
```

SIEM export:

```bash
arctl audit-summary receipt.json --siem-format ecs
arctl audit-summary receipt.json --siem-format otel
arctl audit-summary receipt.json --siem-format cef
```

## Python API

```python
from agentauth.receipts.compliance import export_compliance_mapped, export_siem_ecs

mapped = export_compliance_mapped(bundle, "eu-ai-act")
assert mapped["completeness"]["complete"]

ecs = export_siem_ecs(bundle)
```

## SIEM formats

| Format | Output | Notes |
|--------|--------|-------|
| `ecs` | JSON | Elastic Common Schema; `@timestamp`, `event.*`, `agent_receipts.*` |
| `otel` | JSON | OpenTelemetry log record shape with `attributes` |
| `cef` | string | ArcSight CEF single-line event |
