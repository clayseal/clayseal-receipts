The current Identity Service issues RS256 JWTs based on API key authentication. In production, this becomes a SPIRE deployment that issues SVIDs (SPIFFE Verifiable Identity Documents) — X.509 certificates whose Subject Alternative Name encodes a URI like spiffe://agentauth.io/customer/acme/agent/finance.

The attestation flow works like this: when an agent workload starts, the SPIRE Agent running on the same node intercepts the registration request. It uses node attestors to verify the environment — AWS instance identity documents, Kubernetes service account tokens, GCP metadata server attestation — and workload attestors to verify the process — container image hash, Kubernetes pod labels, process UID. Only if both checks pass does the agent get an SVID. An agent can't claim to be a finance type just by having credentials; it has to prove it's running in an environment that's been pre-approved for that agent type.
